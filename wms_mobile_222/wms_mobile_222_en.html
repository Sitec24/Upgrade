5.0.222 Release Notes

  • The logging system has been revised. Information about the operation, page, and current user has been added. Duplicate entries have been removed, and all records have been standardized.
  • The scanning mode configuration has been redesigned. The Barcodes tab now includes three scanning modes: Auto, Keyboard Event, and Broadcast Intent.
    Auto – in this mode, the scanner connection parameters are configured automatically. The Auto mode is available only on devices from manufacturers with which integration has been implemented (see the list below). It is recommended to reset the mobile terminal scanner settings to factory defaults after enabling this mode.
    List of manufacturers whose devices are integrated: Honeywell, Urovo, Point mobile, CipherLab, Zebra, Newland, M3, Meferi, iData.
    Broadcast intent – when this mode is selected, the integration mechanism with the built-in barcode scanner is used, which sends scan results as a Broadcast Intent message within the Android system.
    Keyboard event – in this mode, the scanner operates by simulating keystrokes on the device keyboard.
    When using the Broadcast Intent or Keyboard Event modes, the corresponding settings must be enabled on the mobile terminal itself.
  • Fixed a bug where it was impossible to scan connection settings via the camera on CipherLab mobile terminals.
  • Fixed an issue where connection instability could cause errors – for example, after completing one task on the mobile terminal, the next task was automatically started.
  • Fixed a bug that caused connection problems with 1С:WMS ver. 4.x.
  • Fixed other errors.
