Изменения в релизе 5.0.222

  • Пересмотрена система логирования. Добавлены данные об операции, странице и текущем пользователе. Убраны дублирующие строки, записи приведены к единому стандарту.
  • Переработана настройка режима сканирования. На закладке «Штрихкоды» доступны 3 режима сканирования: Auto, Keyboard Event и Broadcast Intent.
    Auto – при данном режиме, автоматически устанавливаются параметры подключения к сканнеру. Режим Auto доступен только на оборудовании производителей, с которыми реализована интеграция (перечень ниже). Рекомендуется после установки данного режима сбросить настройки сканера ТСД до заводских.
    Перечень производителей с оборудованием которых произведена интеграция: Honeywell, Urovo, Point mobile, CipherLab, Zebra, Newland, M3, Meferi, iData.
    Broadcast intent – при выборе данного режима будет использоваться механизм интеграции со встроенным сканером штрихкодов, подразумевающий передачу результатов сканирования как сообщение Broadcast intent в систему Android.
    Keyboard event – при данном режиме работа сканера происходит через имитацию нажатия клавиш на клавиатуре устройства
    При использовании режимов Broadcast intent и Keyboard event необходимо включить соответствующие настройки в самом ТСД.
  • Исправлена ошибка. Невозможно было произвести сканирование настроек подключения через камеру на ТСД CipherLab.
  • Исправлена ошибка, когда при плохой связи могли происходить ошибки. Например, при завершении задачи на ТСД, автоматически выполнялась следующая задача.
  • Исправлена ошибка, когда возникали проблемы при подключении к 1С:WMS Логистика. Управление складом ред. 4.x.
  • Исправлены прочие ошибки.
